【Vim】（一）基本操作

发表于 2019-05-14 | 分类于 Vim学习笔记

前言

在开发的路上换了不少编辑器，都没有我想要的操作感，直到我看到了vim。

想法

我认为工具的学习曲线是通过使用逐步扩展的，根本没有必要一开始就记忆一些繁琐的命令，以自己的编码风格去学习和拓展是最平滑，也是最有效的。

下载

去Vim官方网站下载，点击左侧栏的Download进入下载页面，选择合适的版本下载安装，默认选项下一步就可以了。
我下载的是gvim8.1版本，可以通过:version查看当前正在运行的vim的所有信息。

Vim的三种模式

首先要知道vim的三种模式，通过练习体会，这样会理解得更深。

  1. 命令模式（Command mode）
    此状态下敲击键盘动作会被vim识别为命令，而非输入字符，可以进行快捷的操作。
  2. 输入模式（Insert mode）
    有很多方式可以进入输入模式，这样我们就可以编辑代码了。
  3. 底线命令模式（Last line mode）
    通过:（英文冒号）进入底线命令模式，可以进行编辑之外的事情，例如保存文件，修改编辑器的设定等。
Vim三种模式 作用
命令模式 快捷键的操作
输入模式 输入文本
底线命令模式 编辑之外的设定

基本操作

1. 打开工作区

日常不就是先打开编辑器，并且打开所在的工作区吗，同理，打开vim，通过:e

2. 编辑

  • 光标的移动
    通过h j k l。移动光标是最常用的编辑器命令，这些键位在你右手本位。
    h代表向左，j代表向下，k代表向上，l代表向右。

    简单的记忆方法：j向下伸的长，所以向下；k向上冒的长，所以向上

  • 输入

进入输入模式的方法 效果
i 在光标所在的字符前插入 (insert)
I 在光标所在那一行开头插入
a 在光标所在的字符后插入 (append)
A 在光标所在那一行最后插入
o 在光标所在的字符下方建立新空行
O 在光标所在的字符上方建立新空行
  • 修改
    删除一个字符，把光标移动到其上面输入x
    删除一整行，输入dd
    经常用的ctrl+C撤销变成了u
    经常用的ctrl+Y回退变成了ctrl+r

    U大写的U是特殊版本的撤销命令，称为行撤销，即撤销所有在最近编辑的行上的操作。

JSX简介

1. 在JSX中使用表达式

  • 书写JSX的时候推荐外面扩上一个小括号
    为了增强代码可读性，JSX会有换行和缩进，带上括号，防止分号自动插入的bug
  • JSX中的表达式包含在大括号里面

2. JSX属性

这里挂载的属性就是一个变量和非变量的问题了，若是一个变量，则拿大括号包裹起来，若是一个确定的值，则用引号包裹。

注意：JSX的特性更接近Js而不是Html，所以ReactDOM用camelCase小驼峰命名定义属性而不是HTML属性
eg. class => className tabindex => tabIndex

3. JSX防注入攻击

所有内容在渲染之前都被转换成了字符串，可以有效防止XSS攻击。

4. JSX代表Objects

用JSX语法写的HTML标签，其实最后会被React通过方法转化为一个对象，与此同时，React通过读取这些对象来构建DOM并保持数据内容一致。

页面元素的渲染

1. 根节点

在渲染的html中，在此div中的所有内容都将由React DOM来管理。

用React开发一般只会定义一个根节点，若是在一个已有项目中引入React，则需要在不同的部分单独定义React根节点。

2. React元素都是不可变的

当元素被创建之后，无法改变其内容或属性，现阶段的唯一方法是创建一个新的元素，然后将它传入ReactDOM.render()方法中。

3. React只会更新必要的部分
